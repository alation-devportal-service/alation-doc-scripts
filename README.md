# 🛠️ Alation Documentation & API Utility Scripts

Welcome to the central repository for Alation's documentation, metadata, and OpenAPI utility scripts. This collection of tools is designed to automate repetitive tasks, validate URLs, manage Sphinx reStructuredText (`.rst`) metadata, and streamline publishing OpenAPI specifications to ReadMe.

---

## 📋 Prerequisites & Setup

Most of the Python scripts share a common set of dependencies. It is recommended to use a Python virtual environment.

**1. Install Python Dependencies:**

```bash
pip install requests beautifulsoup4 pandas pyyaml python-dotenv
```

2. **Install Node.js Dependencies (For OpenAPI Scripts):**
   
The OpenAPI scripts require `Node.js` and `npx` to be installed on your system to run external validation CLIs.

```bash
npm install -g swagger-cli @redocly/cli rdme
```

3. **Environment Variables:**

For scripts that interact with ReadMe, create a `.env` file in the root directory:

```bash
README_API_KEY=your_readme_api_key_here
```

## 🗂️ Script Catalog & Usage Guide

### 1. URL & Sitemap Utilities

#### url_extractor.py

Parses a local HTML sitemap (`sitemap.html`), extracts all relative links, converts them to absolute URLs, and concurrently checks their live status (`OK` vs `404`).

- **Input:** A local `sitemap.html` file (you will need to update the `sitemap_file_path` variable inside the script to point to your file).

- **Outputs:** Generates three files: `page_level_urls.txt`, `section_level_urls.txt` (URLs with # fragments), and `404_error_urls.txt`.

##### Usage:

```bash
python url_extractor.py
```

#### batch_url_sorter.py

Takes the output from the URL extractor and groups the URLs into perfectly sized batches based on their top-level directory structure. This is highly useful for dividing large QA or metadata tasks among team members.

- **Input:** page_level_urls.txt (generated by `url_extractor.py`).

- **Outputs:** A new directory called `url_batches/` containing multiple `.txt` files grouped by directory.

##### Usage:

```bash
python batch_url_sorter.py
```

### 2. Sphinx / reST Metadata Tools

#### update_rst_metadata.py

Reads a CSV file containing documentation URLs and corresponding metadata (Topics, User Role, Functional Area, etc.), maps the URLs to your local repository's `.rst` files, and injects a properly formatted Sphinx `.. meta::` directive into the document just below the title.

- **Input:** A CSV file containing a Page URL or URL column, alongside metadata columns.

##### Usage:

```bash
# Standard run (will skip files that already have a meta block)
python update_rst_metadata.py metadata_mapping.csv

# Force overwrite existing meta blocks
python update_rst_metadata.py metadata_mapping.csv -f
```

#### replace_separators.py

A cleanup utility that targets existing `.. meta::` blocks in your `.rst` files. It reads a list of URLs, maps them to local files, and replaces any comma `(,)` separators within the metadata values with semicolon `(;)` separators to ensure proper Sphinx parsing.

- **Input:** A text file containing a list of URLs (e.g., one of the batches from `batch_url_sorter.py`).

##### Usage:

```bash
python replace_separators.py url_list.txt
```

### 3. OpenAPI & ReadMe Upload Workflows

**Note:** validate_upload_oas.py is the most robust version of this workflow, combining Swagger, Redocly, and ReadMe CLI validations.

#### validate_upload_oas.py (and update_openapi_and_upload.py)

Automates the process of pulling the latest Alation engineering repository, copying specific OpenAPI YAML specifications, prepping them for ReadMe (enforcing HTTPS, disabling the default explorer, injecting URLs), validating them against strict OpenAPI standards, and uploading them to the ReadMe dashboard.

- **Validation Options:** Supports Swagger CLI (legacy OAS 3.0), Redocly CLI (OAS 3.1), and the official ReadMe CLI.

##### Usage:

```bash
# Standard workflow: Pulls from Alation repo, preps, and uploads
python validate_upload_oas.py <filename_without_extension> <target_version>

# Dry-run / Validation mode: Preps the file and lets you choose validation checks without uploading
python validate_upload_oas.py <filename> <version> --dry-run

# Local mode: Uses a local YAML file instead of pulling from the Git repo
python validate_upload_oas.py <filename> <version> --local
```

### 4. Google Workspace Tools

#### gdoc2rst.js

A Google Apps Script that converts an active Google Document into a Sphinx-compatible reStructuredText (.rst) file, maintaining headings, lists, inline code snippets, tables, and images. It then emails the converted .rst file and any image attachments directly to your inbox.

##### Setup:

1. Open your Google Doc.

2. Go to **Extensions** > **Apps Script**.

3. Paste the contents of `gdoc2rst.js` and click **Save**.

4. Refresh your Google Doc.

5. Click the newly created **Convert to .RST** menu item at the top of your Google Doc and select the email option.
